# 数据使用与披露说明 —— Yuunagi Dashboard

## 1. 数据来源类型

本系统使用的所有数据均来自**公开可访问**的第三方渠道，具体分类如下：

### 新闻资讯类

| 来源类型 | 具体来源 | 说明 |
|----------|---------|------|
| 新闻网站 RSS Feed | 新华社、人民日报、中国日报、参考消息、央视新闻、联合早报、iDaily、BBC 中文、德国之声中文等 | 公开 RSS/Atom Feed，共 45 个源（14 国内 + 12 国际 + 19 科技/工程） |
| 科技/工程博客 RSS Feed | MIT 科技评论、Solidot、橘鸦日报、GitHub Blog、AWS Blog、Cloudflare Blog 等 | 已并入科技类别（feeds.yml），共 19 个源 |
| 财经媒体 RSS/API | 华尔街见闻、央视财经、界面新闻、第一财经周刊、21 世纪经济报道、财新网、叶檀财经、经济学人、华尔街日报等 | 公开财经内容，共 10 个金融专用源（代码中硬编码） |

### 金融行情数据类

| 来源类型 | 具体来源 | 数据内容 |
|----------|---------|---------|
| 财经数据 API | 新浪财经 HQ API | A 股/港股/美股指数 + 商品（黄金/原油/美元指数）实时行情 |
| 加密货币 API | CoinGecko API（首选） | BTC/USD 价格 |
| 加密货币 API | Gate.io API（后备） | BTC/USDT 价格 |
| 金融数据 API | 东方财富 | 韩国 KOSPI 指数 |
| 金融数据 API | Yahoo Finance | 美债 10Y 收益率（^TNX） |
| 金融数据 API | Tushare | 申万行业指数（sw_daily）+ 沪深港通资金流向（moneyflow_hsgt） |

### 数据来源分类汇总

| 大类 | 来源数量 | 类型 |
|------|---------|------|
| 公开网站 RSS Feed | 45 个 | 新闻资讯 |
| 财经数据 API | 1 个（新浪） | 金融行情 |
| 加密货币 API | 2 个（CoinGecko + Gate.io） | 金融行情 |
| 金融数据 API | 3 个（东方财富 + Yahoo Finance + Tushare） | 金融行情 |

**所有数据来源均为公有领域或公开 API，不包含以下任何类型的数据**：
- 交易所非公开数据或内部数据
- 付费数据库的专有数据
- 需要特殊授权或协议的受限数据
- 用户上传或用户生成的内容

## 2. 数据使用方式

| 使用方式 | 适用数据 | 具体描述 |
|----------|---------|---------|
| **检索** | 全部新闻 RSS | 通过关键词匹配过滤金融专栏的相关新闻 |
| **分析** | 全部金融行情 | 指数涨跌幅计算、板块强弱判断、品种间相关性分析（Pearson） |
| **摘要生成** | 全部新闻 RSS | RSS 标题和摘要作为 LLM prompt 的输入，生成中文摘要 |
| **产品展示** | 全部数据 | 生成的摘要和行情图表通过 GitHub Pages 公开展示 |
| **去重记录** | 新闻 URL / 标题 hash | 用于跨天去重，仅存储 hash 值和日期，不存储原文 |

### 明确未涉及的使用方式

- **训练**：数据不用于训练或微调任何机器学习模型
- **微调**：不用于模型微调
- **回测**：当前不做历史回测（预留功能）
- **商业用途**：本项目为非商业开源项目，不涉及商业目的
- **再分发**：不重新分发原始数据，仅展示经过 LLM 加工后的摘要（≤60 字）+ 保留原文链接

## 4. 个人信息与敏感数据声明

### 是否包含个人信息

**不包含。** 本系统处理的所有数据均为：

- 新闻媒体的**公开发布内容**（标题、摘要、链接）
- 金融市场的**公开行情数据**（指数价格、涨跌幅）

系统不收集、不存储、不处理以下任何类型的数据：
- 个人身份信息（姓名、身份证号、电话号码、地址等）
- 用户行为数据（浏览记录、点击行为、偏好等）
- 用户生成内容（评论、上传文件等）
- 任何形式的个人隐私数据

### 是否包含敏感信息

**不包含。** 系统不涉及：
- 国家秘密或政府非公开信息
- 商业机密或未公开的商业数据
- 未公开的上市公司财务数据或重大事项
- 任何需要授权才能获取的非公开信息

### 是否包含第三方受限数据

**不包含。** 所有数据来源的使用均遵循以下原则：
- RSS Feed：均为网站主动提供的公开聚合内容，遵循 robots.txt 和 RSS 协议约定
- 金融 API：均使用公开接口或免费层，遵循各平台 API 使用条款
- 不进行高频抓取（每日一次），不绕过任何访问限制
- 不使用任何需要付费授权但未授权的数据

## 5. 数据合规性声明

1. **合法使用权确认**：参赛者确认对作品中所使用的全部数据拥有合法使用权。所有数据均来自公开渠道（公开 RSS Feed、公开 API 接口、公开金融行情数据），不涉及任何非法获取、未经授权访问或违反第三方服务条款的行为。

2. **数据来源合规**：
   - RSS 内容遵循各源网站的 RSS 协议和 robots.txt 约定
   - CoinGecko API 使用公开免费层，遵循其 API 使用条款
   - Gate.io API 使用公开行情接口
   - 新浪财经 API 使用公开行情查询接口
   - 东方财富 API 使用公开数据接口
   - Yahoo Finance 使用公开行情数据
   - Tushare 使用注册用户免费层，遵循其数据使用协议

3. **数据使用合规**：
   - 新闻内容仅用于检索、摘要生成和产品展示，保留原文链接指向源站
   - 不重新发布、不全文转载、不用于商业目的
   - 行情数据仅供信息参考，所有页面均注明"不构成投资建议"

4. **无内幕信息与未公开数据**：系统不包含任何内幕信息（《证券法》第 51 条定义的內幕信息）、泄露数据或未公开的市场数据。所有金融数据均为交易所或数据提供商公开披露的行情信息。

5. **无个人信息**：系统不收集、不存储、不处理任何个人信息（依据《个人信息保护法》第 4 条定义），不涉及用户数据的任何处理。

6. **开源合规**：本作品基于 MIT/Apache 2.0 等开源许可证的第三方组件构建，所有依赖均已在其许可证允许范围内使用，详见"开源与第三方材料说明"。

7. **免责声明**：
   - 本系统展示的行情数据和市场分析仅供参考，**不构成任何投资建议**
   - 新闻摘要由 AI 自动生成，可能存在事实性偏差，建议点击原文链接核实
   - 因数据来源方变更接口、限流或关停导致的短暂数据缺失，系统已通过容错机制处理
   - 因数据使用不当产生的法律责任，由使用者自行承担

8. **数据保护措施**：
   - 原始数据不公开发布（仅在 GitHub Actions CI 环境中临时处理）
   - 公开发布的内容仅为 LLM 加工后的短摘要（≤60 字）和行情图表
   - RSS 原文链接保留，用户可访问原始来源获取完整信息
   - 去重记录中的 hash 值不包含可逆推出原文的足够信息

## 附录：数据字段说明

### NewsItem（新闻条目）核心字段

| 字段 | 类型 | 说明 | 是否公开 |
|------|------|------|---------|
| `title` | string | 新闻标题 | 是（公开摘要中展示） |
| `link` | string | 原始链接 | 是（指向源站） |
| `summary` | string | LLM 生成的中文摘要（≤60 字） | 是 |
| `source` | string | 来源名称 | 是 |
| `category` | string | 大类（国内/国际/科技/工程/要闻） | 是 |
| `sub` | string | 子类（LLM 分配） | 是 |
| `date` | Date | 发布日期 | 是 |
| `urlHash` | string | URL 的 FNV-1a hash | 否（仅用于去重，不公开） |
| `titleHash` | string | 标题的 FNV-1a hash | 否（仅用于去重，不公开） |

### Quote（行情报价）核心字段

| 字段 | 类型 | 说明 | 是否公开 |
|------|------|------|---------|
| `name` | string | 品种名 | 是 |
| `price` | number | 当前价格 | 是 |
| `changePct` | number | 涨跌幅（%） | 是 |
| `change` | number | 涨跌额 | 是 |
| `open/high/low` | number | 日内 OHLC（仅 A 股） | 是 |

### 状态文件结构（不公开发布，仅在 CI 中使用）

```
data/
├── seen-news.json          # 新闻去重记录：{ urls: {hash: date}, titles: {hash: date} }
├── seen-finance.json       # 金融去重记录（同上结构）
├── recent-events-news.json  # 近期事件标题：{ jobs: { name: { date: [titles] } } }
└── recent-events-finance.json # 同上
```

这些文件仅存储 hash 值和日期/标题文本，不包含任何原始新闻内容、个人信息或敏感数据。
