# Yuunagi Dashboard - 产品说明文档（简版）

> 完整详细版见：`docs/产品说明文档.md`（47,000 字，含源码级技术细节）

## 1. 项目名称

**Yuunagi Dashboard**（个人信息聚合看板）

## 2. 项目简介

Yuunagi Dashboard 是一套**全自动多源信息聚合与 AI 日报生成系统**。系统每日从 45 个 RSS 源和 6 个金融行情 API（新浪财经、CoinGecko、Gate.io、东方财富、Yahoo Finance、Tushare）抓取数据，经 LLM 流水线（DeepSeek API）进行事件合并、中文摘要、要闻提炼，最终生成 12 篇结构化日报（4 时事专栏 + 1 时事汇总 + 6 金融专栏 + 1 金融汇总），通过 GitHub Actions 全自动部署到 GitHub Pages。

**公网 Demo**：https://uynajgi.github.io/yuudashboard/（无需登录）  
**代码仓库**：https://github.com/UynajGI/yuudashboard

## 3. 解决的问题与应用场景

- **信息过载**：45 个源日产 800+ 条标题，系统通过 LLM 精选合并压缩为 50-80 条结构化条目，掌握当日全貌
- **同质化报道**：同一事件被数十家媒体转载——LLM 识别并合并，读者看到的是一条事件而非 N 条重复新闻
- **金融信息碎片化**：A 股/港股/亚盘/美股/商品/加密的行情和新闻分散在不同平台——系统产出 6 个单市场专栏 + 1 个跨市场汇总
- **每日复盘耗时**：手工整理需 1-2 小时，系统全流程自动化，定时 19:00 运行

## 4. 核心功能说明

### 4.1 多源数据采集

系统实现统一的源抽象层（`ItemSource` / `MarketSource`），所有数据源插件式注册：

| 类别 | 来源 | 数量 |
|------|------|------|
| RSS 新闻 | 新华社、人民日报、联合早报、MIT 科技评论、橘鸦日报等 | 45 源 |
| 金融行情 API | 新浪财经、CoinGecko/Gate.io、东方财富、Yahoo Finance、Tushare | 6 个 |
45 源并发抓取（`Promise.all`），单源失败不阻塞。行情数据在线拉取不存本地。

### 4.2 三层去重 + LLM 语义去重

- **第一层**：URL FNV-1a hash，拦截跨源转载同一 URL
- **第二层**：标题归一化 hash，拦截完全相同的标题
- **第三层**：标题 bigram Jaccard 相似度 > 0.7，拦截同事件不同措辞
- **第四层**：LLM 跨天语义去重，判断"延续报道"vs"全新事件"
- **辅助过滤**：URL 日期硬拦截（防 RSS pubDate 造假）、hard cap 50 条/源（防历史归档撑爆内存）

### 4.3 LLM 流水线加工

| 阶段 | LLM | 职责 |
|------|:---:|------|
| ingest | - | 并发抓取 → 三层去重 → 市场关键词过滤 → 按类别分组 |
| select | ✓ | 识别同事件合并；分配子类；跨天语义去重（含 recent-events 注入） |
| summarize | ✓ | 为每个事件生成 1-2 句中文摘要（≤60 字） |
| tldr | ✓ | 基于摘要生成 4-6 条高层要闻（含数字和趋势判断） |
| digest | ✓ | 阅读专栏日报，综合生成 8-12 条跨领域/跨市场汇总 |

### 4.4 12 篇日报

**时事线（5 篇）**：国内 / 国际 / 科技 / 工程 / 今日要闻汇总

**金融线（7 篇）**：A 股 / 港股 / 亚盘 / 美股 / 商品 / 加密 / 金融市场汇总

**金融专栏内容**：指数表 + chart.xkcd 涨跌柱状图 + **sparkline 7 日走势图** + A 股专属申万行业指数和北向资金 + 分类新闻

**汇总报告**：LLM 综合点评 + 全市场指数对比表 + 全市场涨跌柱状图 + 核心资产表

### 4.5 chart.xkcd 图表与可视化

- **柱状图（Bar）**：各指数/板块当日涨跌幅对比，涨绿跌红
- **Sparkline 走势图（Line）**：资产近 7 天走势折线，数据不足 3 天自动跳过。覆盖指数、资产、BTC、美债等品种
- **双线对比图（Line）**：两品种归一化折线，用于联动观察
- CDN 不可用时静默降级为纯表格（`if(!window.chartXkcd)return` 守卫）

### 4.6 前端展示

Hugo 静态站 + PaperMod 主题。首页卡片瀑布流（含 TL;DR 摘要侧边栏）+ 无限滚动加载。暗色模式自适应、KaTeX 公式、Lenis 平滑滚动、Turbo 瞬时切换。

## 5. AI 使用方式与技术方案

### 5.1 AI 能力使用清单

对照比赛要求，逐项说明本作品对各类 AI 能力的使用情况：

| AI 能力 | 是否使用 | 在作品中的具体体现 |
|---------|:-------:|-------------------|
| **大语言模型（LLM）** | ✅ | DeepSeek V4 Flash，承担全部语义理解任务（事件合并、摘要生成、要闻提炼、跨专栏综合） |
| **智能体（Agent）** | ✅ | 3 个 Agent Profile（板块点评/资产聚焦/联动观察），LLM 自主决定调用哪些工具、调用顺序和次数，多轮循环直到产出结论 |
| **多模态模型** | ❌ | 当前仅使用文本理解，未涉及图像/语音/视频模态 |
| **RAG（检索增强生成）** | ✅ | `recent-events.json` 机制：每次渲染后存储已发布事件标题，下次 select 时自动注入 prompt，LLM 据此判断当前候选是"延续报道"还是"全新事件" |
| **工具调用（Function Calling）** | ✅ | 5 个 Agent 工具（`get_market_snapshot` / `get_history_series` / `get_finance_news` / `compute_correlation` / `find_correlations`），原生 DeepSeek function calling，零框架依赖 |
| **自动化工作流** | ✅ | GitHub Actions cron 定时触发 → 并行抓取 → LLM 多阶段流水线 → 模板渲染 → git push → Hugo build → GitHub Pages 部署，全链路无人值守 |
| **模型微调** | ❌ | 使用 DeepSeek 官方 API 直接调用，未进行微调。所有质量控制通过提示词工程实现 |
| **提示词工程** | ✅ | 6 个提示词模板（daily/finance × select/summarize/tldr），约 10,000 字，经多次迭代精炼；含示例驱动的质量对齐（"良好例子"vs"不好例子"对比）、负面约束、JSON schema 严格约束 |
| **文本理解** | ✅ | 核心依赖：从新闻标题/摘要中理解事件主体、核心事实、因果关系，判断事件合并和子类归属 |
| **语音/图像理解** | ❌ | 当前未涉及 |

### 5.2 AI 定位

**AI 是本作品的核心功能，而非辅助功能。** 去掉 LLM 流水线后，系统退化为纯 RSS 聚合器——只展示原始标题列表，丧失事件合并、中文摘要、要闻提炼、跨专栏综合等全部高价值能力。AI 不是锦上添花，而是系统的认知中枢。

### 5.3 模型

**DeepSeek V4 Flash**（通过 DeepSeek API 调用，兼容 OpenAI SDK 格式）。

### 5.4 AI 核心作用（6 项不可替代职责）

1. **事件合并（select）**：识别不同措辞的标题是否指向同一事件——纯规则不可能做到
2. **中文摘要（summarize）**：从多源信息中提炼 1-2 句核心事实（生成式摘要）
3. **要闻提炼（tldr）**：在摘要基础上提升抽象层次，点出"事件+数字+影响"
4. **跨天语义去重**：判断"延续报道"vs"全新事件"（RAG 模式：检索近期标题注入 prompt）
5. **跨专栏/市场汇总（digest）**：阅读多篇专栏后综合生成汇总
6. **Agent 自主分析（可选）**：LLM 通过 Function Calling 自主调用 5 个工具完成分析任务

提示词体系约 10,000 字，按（分线，阶段）矩阵组织，每阶段输出严格 JSON。

## 6. 数据来源与数据处理

### 数据来源

| 类型 | 来源 | 说明 |
|------|------|------|
| 新闻 RSS | 45 个公开 RSS 源 | 新华社、人民日报、联合早报、MIT 科技评论、Solidot、橘鸦日报等 |
| 财经 API | 新浪财经 HQ API | A 股/港股/美股指数 + 商品行情（GBK 编码，三格式自动检测） |
| 加密 API | CoinGecko → Gate.io fallback | BTC/USD，多源容错 |
| 其他 API | 东方财富、Yahoo Finance | KOSPI、美债 10Y |
| 金融数据 | Tushare | 申万行业指数 + 北向资金（可选） |
### 数据处理

- RSS：XML 解析 → 8 步 HTML 实体解码 → 聚合 RSS 拆分 → 时间窗过滤 + hard cap 50 条
- 行情：GBK 解码 → 按字段数区分 A 股（32）/港股（6）/美股（4）格式 → 统一 Quote 模型
- 去重：URL hash → 标题 hash → bigram Jaccard → LLM 语义去重

### 数据合规

所有数据均为公开可访问的 RSS Feed 或公开 API。不包含个人信息、内幕信息或非公开数据。RSS 保留原文链接。行情数据仅供参考，不构成投资建议。

## 7. 产品运行方式与演示

### 公网 Demo

**地址**：https://uynajgi.github.io/yuudashboard/  
**登录**：无需登录，完全公开  
**更新**：每日北京时间 19:00 自动更新

### 体验路径

1. 首页 → 文章卡片瀑布流，TL;DR 摘要直接展示
2. 时事专栏 → 按子类分组的新闻列表（标题 + LLM 摘要 + 原文链接）
3. 金融专栏 → 指数表 + chart.xkcd 柱状图 + sparkline 走势图 + 申万行业 + 北向资金 + 分类新闻
4. 汇总报告 → LLM 综合点评 + 全市场指数对比 + 图表 + 跳转链接
5. **验证 AI** → 点击任意新闻标题跳转原文，对比 LLM 摘要的准确性

### 本地运行

```bash
git clone --recurse-submodules git@github.com:UynajGI/yuudashboard.git
cd yuudashboard/scripts
npm ci
cp .env.example .env  # 填入 DEEPSEEK_API_KEY
node v2/index.js --job=daily-news-domestic
```

### 部署到自己仓库

Fork → 改 `hugo.toml` 的 `baseURL` → Settings → Secrets 添加 `DEEPSEEK_API_KEY` → 启用 Actions 和 Pages → 手动触发 Generate Reports

## 8. 项目创新点

1. **LLM 驱动的多阶段信息精炼流水线**：按认知层级分阶段加工（合并→提取→提炼→综合），对应人类编辑完整流程
2. **三层机械去重 + LLM 语义去重**：廉价去重拦截 99%，昂贵 LLM 仅处理 1% 边界情况
3. **"先分后总"的报告体系**：专栏独立生成（窄领域低噪音），汇总整合已提炼信息（高层次跨领域）
5. **原生 Function Calling Agent**：约 110 行实现完整 Agent 循环，5 个工具，零框架依赖
6. **chart.xkcd 手绘金融图表 + sparkline**：涨跌柱状图 + 7 日走势折线，CDN 不可用时静默降级
7. **全流程 CI/CD 无人值守**：cron 定时 → 并行管线 → artifact 合并 → 一次 push → 自动部署
8. **插件式架构**：加数据源 = 一个文件 + 一行注册，加报告 = jobs.yml 加一行配置

## 10. 风险提示与局限性

- **LLM API 依赖**：核心加工依赖 DeepSeek API，不可用时日报中断。已通过容错处理确保单次失败不阻塞次日
- **数据源稳定性**：RSS/API 均为第三方服务，单源失败不影响整体（单源 try-catch）
- **LLM 输出质量波动**：摘要可能偶有偏差，所有条目保留原文链接可交叉验证
- **仅支持中文**：prompt 和输出均为中文
- **仅支持 DeepSeek**：尚未适配其他 LLM 后端
- **单用户场景**：不支持多用户和个性化
