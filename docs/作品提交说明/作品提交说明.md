## 作品提交说明

参赛者须在规定截止时间前提交作品材料。作品提交以压缩包形式进行，压缩包命名格式建议为：

## 姓名-项目名称.zip

压缩包内应包含产品说明文档、作品展示材料，以及可选的源码或代码仓库链接。参赛者应确保提交材料完整、可打开、可访问，并能够让评委理解作品的核心功能、AI 使用方式、技术实现和实际完成度。

## 一、必交材料

## 1. 产品说明文档

参赛者须提交一份产品说明文档。文档格式不限，可使用 PDF、Markdown、Word、PPT 或其他常见格式。

产品说明文档建议包含以下内容：

1. 项目名称

2. 项目简介

3. 解决的问题或应用场景

4. 核心功能说明

5. AI 使用方式与技术方案

6. 使用的数据来源与数据处理方式

7. 产品运行方式、访问方式或演示方式

8. 项目创新点

9. 当前完成度

10. 风险提示、局限性或未来改进方向

其中，“AI 使用方式与技术方案”应尽量说明 AI 在作品中的具体作用。例如：是否使用大语言模型、智能体、多模态模型、RAG、工具调用、自动化工作流、模型微调、提示词工程、语音/图像/文本理解等能力；AI 是作为核心功能，还是作为辅助功能。

## 2. 作品展示材料

参赛者须选择以下任一方式展示作品：

## 方式一：公网可访问 Demo

如作品已部署至公网环境，参赛者可在产品说明文档中提供可访问链接，并说明必要的使用方式，包括但不限于：

1. Demo 链接

2. 测试账号或登录方式，如有

3. 核心功能体验路径

4. 使用步骤

5. 注意事项

参赛者应确保 Demo 链接在评审期间可正常访问。如 Demo 无法访问、无法登录或主要功能无法使用，可能影响评分。

## 方式二：录屏 Demo

如作品不方便部署公网访问，参赛者可提交录屏文件或录屏链接。录屏应展示作品的主要功能、完整使用流程和核心亮点。

## 录屏 Demo 建议包含以下内容：

1. 项目简介

2. 使用场景说明

3. 核心功能演示

4. AI 能力展示

5. 输入、处理过程和输出结果

6. 项目亮点说明

产品说明可以通过文档呈现，也可以在录屏中通过配音讲解。若主要通过录屏讲解，仍建议提交一份简短文档，便于评委快速了解项目。

## 二、可选材料

## 1. 源码或代码仓库链接

参赛者可自愿提交源码、GitHub 仓库链接、GitLab 仓库链接或其他代码托管链接。

源码不是强制提交项。若参赛者因商业计划、隐私、安全、第三方协议或其他合理原因不希望公开源码，可仅提交公网 Demo 或录屏 Demo，以及产品说明文档。

如提交源码，建议在压缩包或代码仓库中包含 README，说明项目启动方式、依赖安装方式、环境变量配置、主要目录结构和运行命令。

## 2. 补充材料

参赛者可根据作品情况自愿提交以下补充材料：

1. 系统架构图

2. 产品截图

3. API 文档

4. 数据字段说明

5. 模型或提示词设计说明

6. 测试样例

7. 回测结果或实验结果

8. 其他有助于评委理解作品的材料

## 三、数据使用与披露要求

参赛者可使用公开数据、自行整理数据、第三方 API 数据、付费数据、自有数据或模拟数据完成作品，但须确保数据获取、使用和展示符合相关法律法规、平台规则、数据服务协议及授权范围。

如参赛者使用的数据不便公开，可不提交原始数据集或完整数据内容，但须在产品说明文档中说明以下信息：

1. 数据来源类型，例如公开网站、交易所公告、财经数据 API、新闻资讯、公司财报、用户自行整理数据、模拟数据等；

2. 数据使用方式，例如用于检索、分析、回测、训练、微调、评估或产品展示；

3. 数据时间范围、市场范围或样本范围的概括说明；

4. 是否包含个人信息、敏感信息、非公开信息或第三方受限数据；

5. 数据合规性声明，即参赛者确认其对相关数据拥有合法使用权，且作品展示未违反数据来源方的使用条款或授权限制。

参赛者无需公开商业敏感数据、付费数据、个人隐私数据或其他不适合公开的数据内容。若无法公开原始数据，建议提供脱敏样例、字段说明、数据处理流程或

模拟数据，以便评委理解作品逻辑。

未公开原始数据不影响参赛资格，但可能影响作品的可验证性、复现性。

参赛者不得使用非法获取的数据、未经授权的个人信息、内幕信息、泄露数据，或违反第三方平台服务条款的数据。因数据来源、授权或使用不当产生的法律责任，由参赛者自行承担。

## 四、开源与第三方材料说明

参赛作品须为参赛者本人在本次活动期间完成或显著完善的原创作品。

如作品基于开源项目、第三方框架、公开代码、模板项目、外部组件、模型服务或 API 服务进行开发，须在产品说明文档中注明来源，并说明本作品在其基础上的新增功能、改造内容或创新点。

不得直接提交未做实质改造的开源项目、模板项目或他人作品。

## 五、提交与评审说明

参赛者应在规定截止时间前完成提交。逾期提交、材料缺失、文件无法打开、链接无法访问、录屏无法播放，或作品主要功能无法判断的，可能影响评分。

评委将主要根据作品的实际完成度、AI 使用合理性、问题定义清晰度、产品可用性、技术实现质量、创新性和展示效果进行综合评估。

提交公网 Demo 的作品，评委将优先通过 Demo 链接体验核心功能；提交录屏Demo 的作品，评委将根据录屏内容和产品说明文档进行评估。

作品应尽量体现可运行、可展示、可解释的完整闭环。仅提交概念说明、纯 PPT构想，或缺少实际功能展示的作品，可能影响评分。
